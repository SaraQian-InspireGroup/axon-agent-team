# 伊利奶粉事业部「分仓补货计划」业务解读

## 一、业务背景

### 1. 业务链路

伊利奶粉事业部的分仓补货业务，覆盖从生产工厂、基地仓、销售分仓，到线上线下渠道客户的供应链补货链路。

整体链路可以概括为：

```text
工厂 / 基地仓 → 销售分仓 → 电商及经销商 → 终端客户
```

业务涉及的上游节点包括：

- 8 个国内工厂
- 1 个 ODL 工厂
- 4 个基地仓
- 多个区域销售分仓
- 线上电商渠道与线下经销商渠道

其中，基地仓包括天津基地仓、呼市基地仓、杜蒙基地仓、武汉基地仓等；销售分仓包括合肥、天津、广州、郑州、成都、武汉、呼市、济南、柳州等销售仓。

### 2. 组织边界
- **组织范围**：伊利奶粉事业部
- **业务范围**：伊利奶粉事业部所属生产工厂到各分仓的补货业务
- **商品范围**：伊利奶粉事业部经营的所有在售商品，不包含促销品及礼盒
- **覆盖供应链网络节点**：工厂仓、区域分仓
- **渠道范围**：线上电商、线下经销商

### 3. 业务本质和目标

> 未来需要建立一套面向奶粉事业部的分仓补货计划机制，使生产、物流、销售分仓和渠道需求之间形成更高效、更准确、更可控的协同关系。

奶粉业务具有较强的供应链复杂性：

- 渠道多，包括线上电商、KA 客户、经销商、母婴、商超、特渠等
- 仓网多，包括工厂仓、基地仓、区域分仓
- 商品对日期和新鲜度敏感
- 区域需求波动明显
- 缺货会影响订单满足和客户体验
- 库存过高会影响周转和新鲜度

因此，分仓补货计划既要解决“货够不够”的问题，也要解决“货在哪里、是否新鲜、是否在正确的仓、能否及时满足订单”的问题。

## 二、业务现状与潜在问题

### 1. 当前业务流程现状

当前分仓补货计划主要涉及三个业务板块：

```text
销售 → 物流 → 生产供应
```

其中：

- **销售侧**：制定销售计划，调整销售计划，整理区域原始需求
- **物流侧**：负责产销协同、生产主计划、排程、基地仓到销售仓的分货计划，以及基地仓到销售仓的补货订单
- **生产供应侧**：负责原奶供应及产能、工厂生产、基地仓入库

现有流程已经具备跨部门协同机制，但关键环节仍然高度依赖人工、Excel 和经验判断。

### 2. 问题一：当前以“推式分货”为主，需求响应偏被动

现有模式主要是“推式分货”为主，即根据生产备货率、库存天数、分配后缺口等指标，由上游向销售仓进行分配。

这种模式的特点是：

- 从生产和库存供给角度出发
- 更关注上游有什么货、如何分出去
- 对下游真实需求变化的响应相对滞后

潜在问题包括：

- 当区域需求快速变化时，分货调整不够及时
- 容易出现部分仓缺货、部分仓积压
- 难以及时反映渠道订单、区域销售节奏和市场变化
- 可能造成偏仓、跨仓履约或库存结构不合理

### 3. 问题二：销售计划整理依赖人工，需求输入颗粒度不足

当前销售计划需要按区域、营销总部归属等维度进行人工整理，再加工到分仓维度。

销售计划整理主要通过 Excel 和调度组完成。同时，分货参考的库存天数计算主要按照日均需求计算，没有充分考虑上下旬趋势。

潜在问题包括：

- 销售计划从区域 / 营销总部到分仓维度的转化成本高
- 人工整理耗时耗力
- 数据口径可能不统一
- 需求趋势识别不足
- 对阶段性促销、渠道波动、区域销售节奏的反映不够敏捷

这说明当前不是简单缺少数据，而是数据需要大量人工加工，并且需求计划的精细化程度仍有提升空间。

### 4. 问题三：分货计划依赖 Excel，系统化和智能化程度不足

当前基地仓到销售仓的分货计划主要依赖 Excel 和调度组处理。

待检、合格、在途、库存等信息也通过 Excel 传递，说明分货计划尚未形成统一、自动化、可追溯的系统能力。

潜在问题包括：

- 数据收集和整理工作量大
- 分货决策依赖人工经验
- 复杂场景下难以兼顾库存、需求、新鲜度、在途、待检、合格库存等多重因素
- 分货过程透明度不足
- 分货逻辑难以沉淀为统一规则
- 决策结果难以复盘和持续优化

尤其在多仓、多渠道、多 SKU 的场景下，单纯依赖 Excel 和经验，很难实现全局最优。

### 5. 问题四：库存监控依赖人工，横向调拨不够及时

当前销售仓和基地仓库存主要由仓内大日期库存人工监控。

当仓内包品过多、库存结构不均、临期风险、仓间不平衡等问题出现时，可能发现较晚，导致销售仓之间横向调拨无法及时兼顾。

潜在问题包括：

- 库存风险发现滞后
- 仓间不均衡难以及时纠偏
- 临期、大日期库存可能影响商品新鲜度
- 部分仓缺货时容易触发跨仓履约
- 物流成本和履约时效受到影响
- 客户订单满足率存在风险

因此，需要从“人工盯库存”升级为“系统监控库存、系统提示风险、系统辅助调拨”。

### 6. 当前整体问题总结

当前分仓补货业务的核心问题可以概括为：

```text
需求输入不够精细
      ↓
分货模式偏推式
      ↓
计划形成依赖人工和 Excel
      ↓
库存监控与调拨响应不及时
      ↓
导致分货效率、分仓有货率、库存周转、新鲜度和订单满足存在提升空间
```

本质上，当前业务仍处于“人找数据、人做判断、人做协调”的阶段，尚未形成系统化、规则化、智能化的分仓补货计划闭环。

## 三、当前业务升级目标

### 1. 总体升级方向

总体目标可以概括为：

> 通过推拉模式设计、销售计划精细化、分仓智能补货优化和销售仓横向调拨，构建更智能、更高效、更均衡的分仓补货计划体系。

不是单纯上线一个工具，而是希望重构分仓补货的业务机制，使其从经验驱动走向系统化、规则化和智能化。

### 2. 专题一：推拉模式设计

当前业务以推式分货为主，未来需要探索“推拉结合”的模式：

- **推式**：基于生产计划、基地仓库存、供应约束、基础铺货需求，由上游主动向下游分配
- **拉式**：基于销售分仓真实需求、渠道订单、区域销售节奏、库存消耗，由下游需求拉动补货

推拉模式设计要解决的问题是：

> 哪些货由上游根据生产和库存主动推，哪些货由销售分仓根据真实需求主动拉，二者如何协同。

这一模式设计将决定未来补货计划的业务逻辑、系统规则和职责边界。

### 3. 第一道防线：销售计划精细化

销售计划精细化是第一道防线，对应专题二。

它的目标是提升需求输入质量，使分仓补货计划建立在更准确、更细颗粒度的销售计划基础上。

主要升级方向包括：

- 将销售计划从粗颗粒度拆解到分仓、区域、渠道、SKU 等更细维度
- 提高计划对上下旬需求趋势、促销节奏和区域波动的反映能力
- 减少人工整理和口径转换
- 提升销售计划作为补货输入的准确性和可用性

销售计划精细化解决的是“需求准不准”的问题。

### 4. 第二道防线：分仓智能补货优化

分仓智能补货优化是第二道防线，对应专题三。

最核心的能力建设方向，目标是从人工 Excel 分货转向系统化、智能化的补货建议。

未来系统需要综合考虑：

- 分仓销售需求
- 分仓库存水平
- 基地仓库存
- 在途库存
- 待检库存
- 合格库存
- 库存天数
- 商品新鲜度
- 订单满足率
- 区域需求变化
- 渠道优先级
- 生产供应约束

该能力要解决的是：

> 如何把有限库存更合理地分配到不同销售仓，并在效率、库存、订单满足和新鲜度之间取得平衡。

### 5. 第三道防线：销售仓横向调拨

销售仓横向调拨是第三道防线。

这说明并不假设第一次分货一定完全准确，而是通过销售仓之间的横向调拨机制进行二次纠偏。

主要目标包括：

- 识别销售仓之间的库存不均衡
- 发现部分仓缺货、部分仓积压的问题
- 在订单履约受影响前及时调拨
- 降低跨仓履约
- 提升整体库存利用效率
- 保障商品新鲜度

销售仓横向调拨解决的是“分货之后如何动态纠偏”的问题。

### 6. 最终目标：四个提高，一个降低

明确提出了目标，即“四个提高、一个降低”。

#### 6.1 提高分货效率

通过系统化规则和智能计算，减少人工整理 Excel、人工判断和跨部门反复沟通，提高基地仓到销售仓的分货效率。

#### 6.2 提高分仓有货率

通过更准确的需求识别和更合理的补货计划，让各销售仓更及时地获得适配货源，减少区域性缺货。

#### 6.3 提高库存周转率

通过优化库存分布，减少货品在不合适仓库中沉淀，提高整体库存周转效率。

#### 6.4 提高新鲜度

奶粉商品对日期敏感，补货计划需要兼顾库存数量和商品日期，减少临期、大日期库存风险，提高终端可售商品的新鲜度。

#### 6.5 降低跨仓履约

通过提前把货放在更接近真实需求的正确仓库中，减少订单发生后再从其他仓补发或调拨的情况。

降低跨仓履约可以带来：

- 降低额外物流成本
- 提升履约时效
- 改善客户体验
- 减少仓间被动协调
- 提高分仓库存配置准确性
