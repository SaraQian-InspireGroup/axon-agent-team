## 一、业务总览

### 1.1 产品定位

OIP 智能补调产品，目标是实现**全国分仓货源调拨的自动化**——通过算法结合月度计划、订单进度和库存情况，自动分配货源，减少人工干预。

### 1.2 核心业务目标

> 在全国范围内，使某一品类的库存保持均衡，所有库存处于安全状态，能够及时满足消费需求。

### 1.3 适用范围

- 当前逻辑覆盖集团 **70%+ 的品项**（线下渠道）
- 电商有另一套独立逻辑
- 70% 的品项可以通用同一套标准

## 二、完整业务流程

### 2.1 流程总览

```
销售计划 → 产销平衡 → 生产排期 → 基地仓入库
                                      ↓
                                正向补货（基地仓 → 销售仓）
                                      ↓
                                横向调拨（销售仓 ↔ 销售仓，临期消化）
                                      ↓
                                分货下发 → 分仓补货单 → 物流执行 → 入库
                                      ↓
                                跟踪监控
```



### 2.2 第一步：全国货源与缺口识别

**使用的工具**：全国库存监控报表

**计算公式**：

```
发货缺口 = 现有库存 + 在途库存 - 未发订单
```

**判断规则**：


| 缺口值 | 含义            |
| --- | ------------- |
| 负数  | 未被满足的需求（存在缺口） |
| 正数  | 订单需求满足后剩余的库存  |
| 零   | 刚好满足          |


**补充说明**：

- 在途库存**已计入**现有库存（一旦上车发货即算在途）
- 在途量放在收货仓的库存中统计
- 报表体现全国各分仓的库存总和 + 分仓维度的未发订单总和



### 2.3 第二步：库存充足度判断

**三个核心变量**：


| 变量   | 定义                    |
| ---- | --------------------- |
| 现库   | 分仓当前实际库存量             |
| 当月累出 | 分仓已发出、满足销售订单的量（实际发生值） |
| 生产计划 | 当月计划生产的量              |


**月度总供应** = 现库 + 当月累出 + 生产计划  
**月度需求** = 销售计划

**判断标准**（按全国维度，非个别仓）：


| 月度供应 vs 月度需求（销售计划） | 判断结果   | 备货率标准                     |
| ------------------ | ------ | ------------------------- |
| 供应 > 需求 **+5%** 以上 | **充足** | 按充足标准（上旬20%→中旬50%→下旬100%） |
| 供应 ≈ 需求 **±5%** 以内 | **持平** | 按持平标准（上旬20%→中旬50%→下旬70%）  |
| 供应 < 需求 **-5%** 以上 | **不足** | 按不足标准（上旬20%→中旬50%，上限更保守）  |


**注意**：5% 阈值为 70% 品项的通用标准，不同品类可能有差异。

### 2.4 第三步：正向补货（基地仓 → 销售仓）



#### 备货率计算

```
备货率 = 库存 ÷ (月度计划 + 出库)
```

> 结合预占、询单看备货率，备货率含预占、询单。



#### 库存充足度分类与备货率标准

根据 **月度供应 vs 月度需求** 的关系，将库存状态分为三类，每类对应不同的备货率标准：


| 库存状态   | 判断条件          | 上旬标准 | 中旬标准 | 下旬（25号）标准 |
| ------ | ------------- | ---- | ---- | --------- |
| **充足** | 库存 > 当月计划 +5% | 20%  | 50%  | **100%**  |
| **持平** | 库存 ≈ 当月计划 ±5% | 20%  | 50%  | **70%**   |
| **不足** | 库存 < 当月计划 -5% | 20%  | 50%  | 见下方说明     |




#### 三种库存状态下的备货率目标明细

**一、库存充足（库存 > 当月计划 +5%）**


| 时间         | 订单进度 ≤ 标准             | 订单进度 > 标准                   |
| ---------- | --------------------- | --------------------------- |
| 上旬（标准=20%） | **40%**               | 订单进度 +20%（不超计划）             |
| 中旬（标准=50%） | 订单 ≤ 标准-20% → **50%** | 标准-20% < 订单 ≤ 标准 → **70%**  |
| 下旬（25号）    | 订单 ≤ 标准-20% → **70%** | 标准-20% < 订单 ≤ 标准 → **100%** |


**旺季特殊标准**：上旬最高 **110%**，当月 110% < 下月 40% 备货率，同时滚动参考下月计划。

**二、库存持平（库存 ≈ 当月计划 ±5%）**


| 时间         | 订单进度 ≤ 标准                         | 订单进度 > 标准                              |
| ---------- | --------------------------------- | -------------------------------------- |
| 上旬（标准=20%） | **20%**                           | 订单进度（不超计划）                             |
| 中旬（标准=50%） | 订单 ≤ 标准-20% → **订单进度 +20%（不超计划）** | 标准-20% < 订单 ≤ 标准 → **订单进度 +20%（不超计划）** |
| 下旬（25号）    | 订单进度 +20%（不超计划） → **100%**        | 订单 > 标准 → **100%**                     |


**三、库存不足（库存 < 当月计划 -5%）**


| 时间         | 订单进度 ≤ 标准                    | 订单进度 > 标准                         |
| ---------- | ---------------------------- | --------------------------------- |
| 上旬（标准=20%） | **15%**                      | 订单进度（不超计划）                        |
| 中旬（标准=50%） | 订单 ≤ 标准-20% → **订单进度（不超计划）** | 标准-20% < 订单 ≤ 标准 → **订单进度（不超计划）** |


> 核心原则：**订单进度超过标准时，在当前订单进度基础上 +20%，但不超过月度计划总量**。不足状态下备货率上限更保守。



#### 特殊规则

1. **试饮装**：按计划 100%，超 100% 按缺口满足，不额外备货
2. **电商专供、渠道定制、零食产品**：单独逻辑
3. **偏远仓单独设计备货率**：考虑运输天数，兰州、新疆、肇东、呼市等仓需单独设计
4. **无计划有订单**：按订单缺口补货



#### 补货量计算示例

以品项 A 为例（库存充足状态）：

- 月度计划：100 件
- 当前库存：50 件
- 备货率：50/（100+出库）≈ 33%
- 当前销售订单：60 件 → 订单进度 60%
- 时间：上旬（标准=20%）
- 判断：订单进度 60% > 标准 20%
- 目标备货率：订单进度 60% + 20% = **80%**（不超计划 100%）
- 需补量 = 目标备货量 - 现有库存 = 补 30 件



#### 调拨决策规则

**优先级**：

1. **时效优先**（当前首要考量）
2. 成本次之（通常与时效正相关，距离短 = 时效好 + 成本低）

**就近原则**：

- 优先从最近的基地仓调拨
- 示例：武汉仓有 7000 件、天津仓有 1055 件 → 天津到天津销售仓距离更近 → 削减武汉分配量，增加天津分配量

**运输方式**：全部陆运（汽车）

#### 进阶考虑因素（80% 通用场景之外的动态浮动）

- 工厂排产计划（直配生产模式）
- 运损率
- 近三个月该品项在该仓的销售情况（预判当月订单能否达百）
- 补货上限限制



### 2.5 第四步：横向调拨（销售仓 ↔ 销售仓）

**触发条件**：产品效期管理

**核心逻辑**：

- 基地仓日期永远是最新的（工厂直产直发），不需考虑效期
- 横向调拨针对**销售仓之间**：某仓消化不理想，需将临期品调往有订单的仓

**决策依据**：

- 产品生产日期/批次效期
- 全国范围内寻找匹配订单
- 优先消化日期较早（临期）的库存

**适用场景**：牛奶等有效期相对较短的品类

### 2.6 第五步：分货下发与补货单

**下发流程**：

1. 调度确认分货方案（缺口仓补足、备货率达标）
2. 点击"分货下发"
3. 系统生成**分仓补货单**，流转至下游销售分仓系统

**补货单包含信息**：

- 品项名称
- 调拨数量
- 毛重、净重、体积
- 调入仓、初始发货仓
- 补货单状态
- 下发时间

**下游分仓操作**：

- 根据补货单的总量（如 80 方 / 10 吨）选择车型
- 考虑是否需要凑车、加量

**时效要求**：

- 当前：15 天内发货（管理机制要求，非系统强制）
- 补货单仅有创建时间，无系统层面的发货截止时间
- 后续计划：结合系统落地，逐步缩短时效

**调度侧边界**：分货下发后，调度侧动作完成。后续由分仓执行，调度转为监控角色。

### 2.7 第六步：跟踪监控

- 监控内容：是否按时发货、运输途中是否有异常
- **当前能力**：
  - 出库信息：有
  - 运输途中追踪：**无**（TMS 系统未集成）
  - 入库信息：有
- 异常处理：事后监控、事后补救（物流延误导致断货时，预警场景有需求但系统未实现）

## 三、推拉模式



### 3.1 拉式补货（主流，70%+ 品项）

- 以**需求驱动**：销售计划 → 拉动生产/调拨
- 参考月度销售计划 + 订单下单进度
- 大部分品项采用此模式



### 3.2 推式补货（少量特殊品项）

- 以**生产驱动**：基地仓为主动方，向销售仓推送
- 适用场景：定制品、专供品、需满足最小生产批量的品项



### 3.3 混合/特殊情况

- 月底强制推货：即使拉式导致某仓库存冗余，仍需按销售计划强行将库存分下去
- 打破规则场景：销售计划不准、实际需求超出预期时，需强行将基地仓库存往下推
- 高销量/波动小的品类偏推式，低销量/波动大的偏拉式（降低生产/库存风险）

## 四、节庆/特殊时期调整



### 4.1 标准调整逻辑

正常标准（库存充足）：上旬 20% → 中旬 50% → 下旬 100%

**旺季调整**（如春节、中秋等重点节日前）：


| 旺季场景  | 调整策略                                             |
| ----- | ------------------------------------------------ |
| 上旬为旺季 | 上旬备货率最高 **110%**，当月 110% < 下月 40% 备货率，同时滚动参考下月计划 |
| 中旬为旺季 | 中旬即需达 **100%**                                   |
| 下旬为旺季 | 下旬 25 号前达 **100%**                               |


**春节位置对应的调整**（农历节日每年阳历日期不固定）：


| 春节在农历位置 | 调整策略              |
| ------- | ----------------- |
| 上旬      | 上月底备货率需达 **200%** |
| 中旬      | 上旬即需达 **100%**    |
| 下旬      | 中旬即需达 **100%**    |




### 4.2 当前痛点

- 农历节日每年阳历日期不固定，无标准化的调整表
- 目前完全**靠经验人工调整**，无系统化工具支持
- 不同品类的调整阈值也有差异

## 五、数据基础



### 5.1 数据来源

- 所有源数据来自**数据中台**
- 基础数据存储在 **OIP 仿真模拟系统**中
- 主数据在品类内已统一，数据层面非常完善



### 5.2 算法运行


| 项目     | 详情                     |
| ------ | ---------------------- |
| 取数时间   | 每天 0:00                |
| 运算耗时   | 约 7 小时（0:00 - 7:00）    |
| 数据预处理  | 占大部分时间（将中台数据加工为模型所需格式） |
| 模型运算本身 | 30 分钟 - 1 小时           |
| 结果交付要求 | 上午 9:00，最晚 10:00 前     |




### 5.3 数据口径

- 业态以内（如奶粉全品类）：口径统一
- 跨业态之间：可能存在差异（但本次场景不涉及）
- 本场景（奶粉智能补调）：数据定义口径统一，无跨品类歧义

## 六、AI/AgentOS 可优化方向



### 6.1 当前算法痛点


| 痛点     | 描述                 |
| ------ | ------------------ |
| 分配不均   | 部分仓库存严重超标，部分仓缺货    |
| 无法打破规则 | 特殊场景（需求超预期）时无法自动调整 |
| 节庆无标准表 | 每年农历不同，调整全靠经验      |
| 物流无监控  | 运输途中无实时追踪          |
| 无经验沉淀  | 历史异常处理经验未被系统化记录    |




### 6.2 AgentOS 结合点


| 能力     | 应用方式                                      |
| ------ | ----------------------------------------- |
| 自然语言规则 | 在本体 action/skill 层用自然语言描述业务规则，Agent 按规则执行 |
| 灵活调参   | 备货率标准、节庆阈值等可通过自然语言修改，无需重写代码               |
| 多方案对比  | 输出多版调拨方案（成本最优 / 时效最优 / 满足率最高），供人工决策       |
| 经验沉淀   | Agent 记录历史异常处理经验，逐步学习优化（AgentOS 已有此能力）    |
| 语音交互   | 未来可通过语音输入需求，Agent 输出补货建议                  |




### 6.3 Agent 执行逻辑示例

```
输入：品项X，某仓库存为负
  ↓
Agent 读取本体中的业务规则（自然语言 skill）
  ↓
判断：库存充足度 → 调拨时效优先 → 就近原则
  ↓
输出：建议调拨方案（从哪个仓调多少件）
  ↓
人工确认/反馈 → Agent 记录经验
```

## 七、关键术语表


| 术语   | 定义                                   |
| ---- | ------------------------------------ |
| 基地仓  | 工厂旁边的仓库，存放新生产的产品，日期最新                |
| 销售仓  | 面向下游销售的分仓，货权在伊利                      |
| 正向补货 | 基地仓 → 销售仓 的纵向调拨                      |
| 横向调拨 | 销售仓 ↔ 销售仓 的横向调拨（临期品消化）               |
| 备货率  | 库存 ÷ (月度计划 + 出库)，反映库存对需求的覆盖程度；含预占、询单 |
| 现库   | 分仓当前实际库存量（含在途）                       |
| 当月累出 | 分仓已发出满足订单的量（实际发生值）                   |
| 月度供应 | 现库 + 当月累出 + 生产计划                     |
| 月度需求 | 销售计划                                 |
| 补货上限 | 基于历史销售预判的单仓最大可接收量                    |
| 在途库存 | 已发货但未入库的库存，计入收货仓的现有库存                |
| 拉式补货 | 以需求（销售计划/订单）为驱动的补货模式                 |
| 推式补货 | 以生产为驱动，主动向销售仓推送的补货模式                 |
| OIP  | 伊利的仿真模拟/计划系统                         |
| TMS  | 运输管理系统（当前未与 OIP 集成）                  |
